An encrypted Arch Linux install you can actually maintain

Most Arch install guides hand you a working system. They don’t hand you one you can fix at 3am when the bootloader misbehaves.

This is the install I run on my own machines. Full disk encryption with LUKS, dynamic volumes via LVM, copy-pasteable commands, and just enough explanation that you’ll know what to undo when something inevitably goes sideways.

I’ve been running Arch for years. The thing that keeps me here isn’t speed, isn’t the AUR, isn’t rolling releases. It’s that every part of the system is something I picked, which means every part is something I can debug later.

Here’s the playbook.

Prerequisites

  • Arch Linux ISO on a bootable USB.1
  • Basic comfort with a root shell. Most of the install happens there.
  • A backup of anything important on the target disk. The partitioning steps erase data.

Connect to the Internet

If you’re on ethernet, the live ISO probably auto-connected. Confirm it before moving on:

ip addr show # You should see an RFC1918 compliant IP address assigned to the ethernet adapter.
ping google.com # Do a quick reality check to confirm your connection.

On Wi-Fi, connect through iwctl:

iwctl # Start the Internet Wireless Daemon (IWD) shell.

[iwd] device list # Check whether your wireless adapter is available.
[iwd] station wlan0 scan # Scan for nearby Wi-Fi networks.
[iwd] station wlan0 get-networks # List the available networks.
[iwd] station wlan0 connect "<Wireless-Network>" # Connect to your network. You'll be prompted for the passphrase.

# Wait a few seconds, then exit IWD with CTRL+D.

ip addr show # Your wireless interface should now have an IP address.
ping google.com # Do a quick reality check to confirm your connection.

Refresh the package index:

pacman -Sy

Slow downloads usually mean a far-away mirror. Fix it before installing anything else:

pacman -S reflector rsync curl
reflector --latest 20 --protocol https --sort rate --save /etc/pacman.d/mirrorlist

Future pacman runs should now be noticeably faster.

Set up the Disk

Make sure UEFI is enabled in your BIOS first.

I’ll use /dev/nvme0n1 throughout. Substitute your actual disk name. On SATA drives it’ll be /dev/sda with the partition suffixes becoming 1 / 2 / 3 instead of p1 / p2 / p3.

Confirm the installer booted in UEFI mode:

ls /sys/firmware/efi/efivars

If that directory exists, you’re good. If it doesn’t, reboot into UEFI mode before continuing.

Partition the Disk

The layout: 1GB EFI, 1GB unencrypted /boot, the rest as encrypted LVM.

fdisk -l # Find the name of the disk you want to install to.
fdisk /dev/nvme0n1 # That's my disk's name; yours might be different.
Command (m for help): p # Show the current partition layout.
Command (m for help): g # Create a new GPT layout. Warning: this erases old partitions.
Command (m for help): n # Create a new partition.

# Create 1 partition of type EFI System (1), size 1G.
# Create 1 partition of type Linux filesystem, size 1G. This will be /boot.
# Create 1 partition with the remaining space, type Linux LVM (30). This will hold the encrypted LVM setup.

Command (m for help): w # Write the partition table.

fdisk -l # Confirm the new partition layout.

# Format the boot-related partitions.

mkfs.fat -F32 /dev/nvme0n1p1 # Format the EFI system partition.
mkfs.ext4 /dev/nvme0n1p2 # Format /boot as EXT4.

EXT4 is the boring choice. It’s simple, well-supported, and fine for most uses. BTRFS, ZFS, and XFS are all valid swaps if you have a specific reason for one.

Encrypt the Disk

Now the encryption:

cryptsetup luksFormat /dev/nvme0n1p3 

# Enter passphrase, you will get this prompted when you boot the system. 
# You do not want to forget this, as you can't un-encrypt your hard drive without it.

cryptsetup open --type luks /dev/nvme0n1p3 cryptlvm # Unlock the partition so we can work with it.

Pick a passphrase you can actually remember. Losing it means losing the disk.

Create Dynamic Partitions

Create LVM volumes inside the encrypted partition:

# Initialize the encrypted partition for LVM.
pvcreate --dataalignment 1m /dev/mapper/cryptlvm

# Create a volume group to contain the logical volumes.
vgcreate volgroup0 /dev/mapper/cryptlvm
lvcreate -L 150G volgroup0 -n lv_root # Assign as much space as you please.
lvcreate -l 100%FREE volgroup0 -n lv_home

# Activate the LVM setup.
modprobe dm_mod
vgscan
vgchange -ay 

# Time to format the created logical volumes.
mkfs.ext4 /dev/volgroup0/lv_root
mkfs.ext4 /dev/volgroup0/lv_home

Mount the Volumes

Mount the new volumes under /mnt:

mount /dev/volgroup0/lv_root /mnt 

mkdir /mnt/home
mount /dev/volgroup0/lv_home /mnt/home

mkdir /mnt/boot
mount /dev/nvme0n1p2 /mnt/boot

mkdir /mnt/boot/EFI
mount /dev/nvme0n1p1 /mnt/boot/EFI

Disk layout is now: EFI partition, unencrypted /boot, and encrypted LVM holding root + home.

Install Arch Linux

Install the base system. Pick intel-ucode or amd-ucode to match your CPU. I’m using Intel below.

pacstrap -K /mnt base linux linux-lts linux-firmware intel-ucode vim base-devel openssh networkmanager lvm2
genfstab -U /mnt >> /mnt/etc/fstab
cat /mnt/etc/fstab # Review the fstab before continuing.

arch-chroot /mnt # Enter the new installation.

Configure the Base System

Timezone and locale first. Swap for yours:

ln -sf /usr/share/zoneinfo/Europe/Madrid /etc/localtime
hwclock --systohc

vim /etc/locale.gen # Uncomment en_US.UTF-8 UTF-8, or your preferred locale.
locale-gen

echo "LANG=en_US.UTF-8" > /etc/locale.conf

Hostname:

echo "my-computer" > /etc/hostname

vim /etc/hosts

Add:

127.0.0.1 localhost
::1       localhost
127.0.1.1 my-computer.localdomain my-computer

Now users, networking, and initramfs support for encrypted LVM. The mkinitcpio.conf step is the one most likely to bite you later, so pay attention to the HOOKS line.

pacman -S linux-headers linux-lts-headers wpa_supplicant wireless_tools netctl dialog sudo

systemctl enable NetworkManager # Start networking automatically after boot.

passwd # Set the root password.
useradd -m -g users -G wheel sergiu # Create your user. Swap 'sergiu' with your chosen name.
passwd sergiu # Set the user's password.

EDITOR=vim visudo # Allow the wheel group to use sudo by uncommenting %wheel ALL=(ALL) ALL

vim /etc/mkinitcpio.conf # Add keyboard, encrypt, and lvm2 before filesystems in the HOOKS array.
mkinitcpio -P

Base system done.

Install a Boot Manager

Now to actually boot the thing. I’m using GRUB. Skip this section if you prefer systemd-boot or rEFInd.

pacman -S grub efibootmgr dosfstools os-prober mtools # Install grub and related packages.

grub-install --target=x86_64-efi --efi-directory=/boot/EFI --bootloader-id=grub_uefi --recheck

mkdir -p /boot/grub/locale
cp /usr/share/locale/en\@quot/LC_MESSAGES/grub.mo /boot/grub/locale/en.mo # Ensure /boot/grub/locale exists and select a different language if desired.

blkid /dev/nvme0n1p3 # Copy the UUID of the encrypted partition.

vim /etc/default/grub

Set:

GRUB_CMDLINE_LINUX="cryptdevice=UUID=<encrypted-partition-uuid>:cryptlvm root=/dev/volgroup0/lv_root"

On SSDs, add :allow-discards to the cryptdevice option for TRIM support. Slight security tradeoff: it leaks block-usage patterns to anyone with raw disk access, which is usually fine for a personal machine.

Generate the GRUB config:

grub-mkconfig -o /boot/grub/grub.cfg

The Moment of Truth

Time to reboot and see whether everything works. Very technical. Very suspenseful.

exit 
umount -R /mnt
cryptsetup close cryptlvm
reboot

If you hit a passphrase prompt and then a login screen, you’re in.

If you hit a kernel panic or a busybox shell instead, the most common culprits are: missing encrypt or lvm2 in the mkinitcpio.conf HOOKS array, a typo in the GRUB UUID, or mkinitcpio -P not having been run after editing the hooks.

Post-Install Tweaks

A few things worth doing immediately:

# Create a swap file.
fallocate -l 16G /swapfile
chmod 600 /swapfile
mkswap /swapfile
echo '/swapfile none swap sw 0 0' | tee -a /etc/fstab

# Install a GUI. Here's one for Gnome users.
sudo pacman -Syu gnome gnome-extra
sudo systemctl enable gdm

What you’ve got now isn’t just a working Arch install. It’s one you’ll still understand the next time something breaks. That’s the whole reason to run Arch in the first place.
